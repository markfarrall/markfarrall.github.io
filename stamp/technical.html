Technical

stamp relies on a number of open source packages to work. At a high level:

  • stamp is an implementation of Electron
  • Integration with the various browsers is provided by Selenium
  • The test files are written in JavaScript and use the Jasmine testing extensions.
  • The tests are run using the Node implementation of Jasmine

If you are comfortable with command line based tools you may prefer to look at Protractor for a more easily scripted approach to automated testing. Or you could take a similar approach to stamp and use the Node version of Jasmine directly.
